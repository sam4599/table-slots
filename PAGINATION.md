# Пагинация — подробное объяснение

## Схема кнопок

```
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  «  │  ‹  │  1  │  2  │  3  │  ›  │  »  │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
  ↑     ↑     ↑                 ↑     ↑
  │     │     │                 │     │
  │     │     └── номера страниц │     │
  │     │                        │     │
  │     └── предыдущая           │     │
  │                              │     │
  └── первая страница            │     └── последняя страница
                                 │
                                 └── следующая
```

---

## Что делает каждая кнопка

| Кнопка | Название | Действие |
|--------|----------|----------|
| **«** | Первая | Переход на страницу 1 |
| **‹** | Предыдущая | Текущая страница минус 1 |
| **1, 2, 3...** | Номер | Переход на выбранную страницу |
| **›** | Следующая | Текущая страница плюс 1 |
| **»** | Последняя | Переход на последнюю страницу |

---

## Логика работы

### Переменные

```
localPosts    = [100 постов]     ← все данные с API
perPage       = 25               ← постов на одной странице
currentPage   = 1, 2, 3, 4      ← текущая страница
pages         = 4                ← всего страниц (100 ÷ 25)
```

### Формула для видимых постов

```
visiblePosts = localPosts.slice(start, end)

где:
  start = (currentPage - 1) × perPage
  end   = start + perPage

Пример (страница 2):
  start = (2 - 1) × 25 = 25
  end   = 25 + 25 = 50
  → посты с индексами 25–49 (26-й по 50-й)
```

### Блокировка кнопок

```
« и ‹ — disabled, когда currentPage === 1
› и » — disabled, когда currentPage === pages
```

---

## Визуальная схема данных

```
Все 100 постов:
[0────24][25───49][50───74][75───99]
   стр.1   стр.2   стр.3   стр.4

«  → всегда на [0────24]
‹  → на шаг назад
›  → на шаг вперёд
»  → всегда на [75───99]
```

---

## Поток данных

```
1. Пользователь нажимает « (первая)
   → goToPage(1)
   → currentPage = 1

2. currentPage изменился
   → visiblePosts пересчитывается
   → slice(0, 25) = первые 25 постов

3. watch(visiblePosts) срабатывает
   → emit('sendPosts', [25 постов])

4. UniversalTable получает
   → visiblePosts = [25 постов]

5. PostsTable получает visiblePosts
   → таблица показывает первые 25 постов
```

---

## Кратко

- **«** — первый набор постов
- **»** — последний набор постов
- **‹** и **›** — переход на соседнюю страницу
- **Номера** — переход на конкретную страницу
